# بوابة طلبات الهوية المؤسسية — أساس

موقع داخلي لاستقبال طلبات الموظفين: بطاقة عمل، بطاقة دخول، توقيع بريد إلكتروني، أو أي طلب آخر. مبني وفق دليل الهوية البصرية لأساس (الألوان، الخطوط، الأنماط).

كل طلب يُحفظ في قاعدة بيانات **Firebase Firestore**، ولفريق التواصل المؤسسي والتسويق **لوحة طلبات** محمية بتسجيل دخول حقيقي (Firebase Authentication) لمتابعة كل طلب وتغيير حالته:

- جديد
- قيد التنفيذ
- مكتمل
- مرفوض / يحتاج تعديل

## محتوى الحزمة

```
index.html      ← الموقع كاملاً (الصفحة + التنسيق + المنطق + الشعار، كل شيء في ملف واحد)
README.md       ← هذا الملف (تعليمات الإعداد فقط، ليس جزءاً من الموقع)
```

> هذا الموقع مبني الآن كملف واحد ذاتي الاكتفاء (`index.html`) — التنسيق (CSS) ومنطق الموقع (JavaScript) وشعار أساس كلها مضمَّنة بداخله، لتسهيل الرفع والإدارة. `README.md` ملف توثيقي فقط يشرح خطوات الإعداد، ولا يُعرَض كجزء من الصفحة — GitHub يعرضه تلقائياً في صفحة المستودع الرئيسية، وهذا سبب وجوده منفصلاً.

## خطوات الإعداد (مرة واحدة فقط)

### 1) إنشاء مشروع Firebase

1. افتح [console.firebase.google.com](https://console.firebase.google.com) وأنشئ مشروعاً جديداً (الخطة المجانية Spark كافية، بدون بطاقة بنكية).
2. من داخل المشروع، اضغط أيقونة **</>** (Web) لإضافة تطبيق ويب، وسمّه مثلاً "Asaas Requests".
3. سيعرض لك Firebase كائن إعداد (`firebaseConfig`) يحتوي على `apiKey`, `authDomain`, `projectId`, إلخ. انسخه.
4. افتح `index.html`، وابحث عن `FIREBASE_CONFIG` (قريباً من بداية كود الـ `<script>` في آخر الملف)، وضع كل قيمة في مكانها بدل `"REPLACE_ME"` — هي موجودة بالفعل مُعبّأة بقيم مشروعك الحالي إن كنت اتبعت الخطوات معي سابقاً.

### 2) تفعيل Firestore

1. من القائمة الجانبية: **Build → Firestore Database → Create database**.
2. اختر أي موقع جغرافي قريب (مثل `europe-west` أو `me-` إن توفر)، وابدأ في **Production mode**.
3. من تبويب **Rules** داخل Firestore، الصق القواعد التالية بالضبط، ثم اضغط **Publish**:

   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /requests/{requestId} {
         allow create: if true;
         allow read, update, delete: if request.auth != null;
       }
       match /config/{docId} {
         allow read: if true;
         allow write: if request.auth != null;
       }
     }
   }
   ```

   - `requests`: أي موظف يستطيع إضافة طلب جديد بدون تسجيل دخول، لكن القراءة والتعديل والحذف محصورة بمن سجّل دخوله (فريق التواصل المؤسسي).
   - `config`: يخزّن إعدادات الصفحة (محتوى النصوص، وحالة الحقول إلزامي/محذوف) — قراءته متاحة للجميع لأن صفحة الموظفين تحتاجه لعرض النموذج، لكن التعديل محصور بالأدمن.

### 3) تفعيل تسجيل الدخول وإنشاء حساب الأدمن

1. **Build → Authentication → Get started**.
2. من تبويب **Sign-in method**، فعّل **Email/Password**.
3. من تبويب **Users**، اضغط **Add user** وأدخل:
   - البريد: `asaasoman98@gmail.com` (هذا البريد مكتوب فعلياً داخل كود `index.html` في ثابت `ADMIN_EMAIL` — إن غيّرته هنا، لا بد أن تغيّره هناك أيضاً بنفس القيمة).
   - كلمة المرور: الكلمة التي تريد أن يستخدمها فريق التواصل المؤسسي لدخول لوحة التحكم (مثل `HMF-ASAAS`).

   > **مهم:** واجهة الموقع تطلب من المستخدم **كلمة المرور فقط** (بدون بريد) لتسهيل الدخول، وتقوم بالدخول داخلياً بهذا البريد الثابت. لذلك لا حاجة لأكثر من حساب واحد لهذا الغرض، ويكفي أن يعرف الفريق كلمة المرور فقط.
   > لتغيير كلمة المرور لاحقاً: لا يوجد في Firebase Console خيار "تعديل" مباشر لكلمة مرور مستخدم موجود، فالطريقة الأضمن هي **حذف هذا المستخدم** من تبويب Users، ثم **Add user** من جديد بنفس البريد وكلمة مرور جديدة.

### 4) رفع الملف إلى GitHub وتفعيل GitHub Pages

1. أنشئ مستودعاً جديداً على GitHub، وارفع `index.html` (و`README.md` إن رغبت) في **جذر المستودع**.
2. من المستودع: **Settings → Pages → Build and deployment → Deploy from a branch** → الفرع `main` والمجلد `/ (root)` → حفظ.
3. بعد دقيقة أو دقيقتين سيظهر رابط الموقع (بصيغة `https://username.github.io/repo-name/`).

### 5) التجربة

1. أرسل طلباً تجريبياً من الموقع — يجب أن يعمل فوراً بدون تسجيل دخول.
2. اضغط **"لوحة التحكم (لفريق التواصل المؤسسي)"** في أسفل الصفحة، وأدخل كلمة المرور التي حدّدتها في الخطوة 3.
3. بعد الدخول، يبقى المتصفح "مسجَّل الدخول" حتى لو أعدت تحميل الصفحة — يمكنك التبديل بين وضع لوحة التحكم ووضع تقديم الطلب بضغط زر **"رجوع لتقديم طلب"** (من داخل اللوحة) أو **"لوحة التحكم"** (من أسفل صفحة الطلب) في أي وقت، بدون الحاجة لإعادة كتابة كلمة المرور كل مرة. زر **"تسجيل الخروج"** هو الوحيد الذي يطلب كلمة المرور من جديد عند المحاولة التالية.
4. ستجد ثلاثة تبويبات داخل اللوحة:
   - **الطلبات**: تأكد من ظهور الطلب التجريبي، وجرّب تغيير حالته من القائمة المنسدلة (جديد / قيد التنفيذ / مكتمل / مرفوض - يحتاج تعديل)، وجرّب زر **حذف** لحذف طلب نهائياً.
   - **إدارة الحقول**: لكل نوع طلب، تستطيع جعل أي حقل إلزامياً أو اختيارياً عبر مربع الاختيار، أو **حذف** الحقل بالكامل من النموذج (وزر **استرجاع** لإعادته لاحقاً).
   - **محتوى الصفحة**: تستطيع تعديل عنوان الصفحة، الوصف التعريفي، وملاحظة الفوتر، ثم **حفظ التغييرات** — تظهر فوراً لأي موظف يفتح الصفحة بعد الحفظ.

## الإدارة من لوحة التحكم

- **حالات الطلب**: جديد، قيد التنفيذ، مكتمل، مرفوض / يحتاج تعديل — تُحدَّث فوراً عند تغيير القائمة المنسدلة لكل طلب.
- **حذف طلب**: زر "حذف" في صف كل طلب، مع تأكيد قبل الحذف النهائي (لا يمكن التراجع).
- **التحكم بالحقول**: من تبويب "إدارة الحقول"، تستطيع لكل نوع طلب:
  - تفعيل/تعطيل خانة "إلزامي" لأي حقل.
  - حذف حقل بالكامل من النموذج العام (لن يراه الموظفون)، مع إمكانية استرجاعه لاحقاً من نفس الصفحة.
- **محتوى الصفحة**: من تبويب "محتوى الصفحة"، تستطيع تعديل عنوان الصفحة والوصف التعريفي وملاحظة الفوتر بدون لمس الكود.

## ملاحظات مهمة

- **الخطة المجانية (Spark)** من Firebase تكفي بسهولة لحجم استخدام داخلي عادي (حدود يومية للقراءة/الكتابة، راجع موقع Firebase للأرقام الحالية).
- `firebaseConfig` (المعرّفات في `index.html`) **ليست سرّية** بطبيعتها — أي موقع يستخدم Firebase من المتصفح يعرضها بشكل علني، والحماية الفعلية تأتي من **قواعد Firestore** و **تسجيل الدخول**، وهي مُفعّلة بالخطوات أعلاه.
- بما أن الدخول الآن بكلمة مرور واحدة فقط (مرتبطة بحساب `ADMIN_EMAIL` الثابت في الكود)، فإن كل من يعرف هذه الكلمة يدخل بنفس الحساب — هذا مقصود لتبسيط الدخول لفريق صغير. إن احتجت حسابات منفصلة لاحقاً (بصلاحيات أو سجلّات مختلفة)، يتطلب ذلك تعديل واجهة الدخول لإضافة حقل البريد من جديد.
- إن نسيت كلمة المرور لاحقاً، اتبع طريقة "حذف المستخدم وإعادة إضافته بكلمة مرور جديدة" المذكورة في الخطوة 3 أعلاه.

## التخصيص

- **الشعار**: الشعار مضمَّن داخل `index.html` كصورة مُرمَّزة (base64)، وليس ملفاً منفصلاً. لتغييره لاحقاً، أرسل لي الصورة الجديدة وسأحدّث لك الملف، أو استخدم أي أداة "Image to Base64" عبر الإنترنت واستبدل النص الطويل بعد `data:image/png;base64,` في وسم `<img class="logo-img">`.
- **نصوص الصفحة الرئيسية والفوتر**: تُعدَّل من تبويب "محتوى الصفحة" داخل لوحة التحكم مباشرة، بدون لمس الكود.
- **حقول كل نوع طلب (إلزامي/حذف)**: تُعدَّل من تبويب "إدارة الحقول" داخل لوحة التحكم.
- **إضافة حقل جديد بالكامل، أو نوع طلب جديد**: يتطلب تعديل كائن `FORMS` داخل كود الـ `<script>` في `index.html` (هذا غير متاح من لوحة التحكم حالياً).
- **الألوان والخطوط**: داخل وسم `<style>` في `index.html`، ضمن `:root { ... }`.
- **حالات الطلب**: داخل كود الـ `<script>` في `STATUS_OPTIONS` (وألوانها في `STATUS_CLASS` ومطابقها في قسم CSS).
